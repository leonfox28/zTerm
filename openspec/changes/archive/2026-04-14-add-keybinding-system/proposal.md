## Why

zTerm 当前已经具备终端、多标签、分屏与设置页基础能力，但核心操作仍然主要依赖鼠标入口，无法形成高效的键盘工作流。根据 `docs/handoff.md` 的当前优先级，快捷键系统是最应该进入下一阶段规划与实现的能力，用 OpenSpec 先定义统一行为可以避免后续快捷键、菜单、终端动作各自散落演进。

## What Changes

- 新增统一的快捷键系统能力，为工作台级命令提供键位声明、注册与触发机制。
- 为首批高频终端操作定义默认快捷键，包括新建终端、关闭标签、切换标签、分屏与聚焦基础工作区动作。
- 规定快捷键处理的作用域与冲突边界，避免在文本输入/终端直连输入场景中错误拦截按键。
- 为后续设置页或 SSH/连接管理功能复用同一套命令-快捷键基础设施预留规范入口。

## Capabilities

### New Capabilities
- `keybinding-system`: 定义全局快捷键注册、命令映射、作用域判断与按键触发行为。
- `terminal-keyboard-actions`: 定义终端工作区内首批终端相关快捷键命令及其用户可见行为。

### Modified Capabilities
- None.

## Impact

- 影响 renderer 工作台层与终端 UI 交互入口，主要涉及 workbench store、terminal store 与主工作区事件分发。
- 影响首批终端命令的交互方式，需要把现有鼠标触发能力补充为可通过快捷键调用。
- 影响后续功能扩展方式：新功能若需要快捷键，应接入统一命令/键位注册体系。
- 不要求本次变更引入新的第三方依赖，优先基于现有 React + Zustand 架构实现。
